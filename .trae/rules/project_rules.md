⚙️ Page Rules

1. Each page.tsx should only import and render the corresponding sections entry:
import UserProfileSections from "@/sections/userProfile";

export default function Page() {
  return <UserProfileSections />;
}

2. ❌ No logic, JSX, or UI should be written inside page.tsx directly.


🧱 Folder Structure Rules:
app/
 ├── ai/
 │   └── layout.tsx        → imports from components/layout/ai/Layout
 ├── panel/
 │   ├── layout.tsx        → imports from components/layout/panel/Layout
 │   └── userprofile/
         └── page.tsx     → imports from sections/userProfile => Use Layout Parent Folder : app/{ai Or panel}/layout.tsx

sections/
 ├── ai/
 │    ├── ChartSection.tsx
 │    ├── SummarySection.tsx
 │    ├── StatsSection.tsx
 │    └── index.tsx
 └── panel/
      ├── profile/
      │    ├── AboutSection.tsx
      │    ├── ProjectsSection.tsx
      │    ├── TagSection.tsx
      │    └── index.tsx

.demo_disabled/
 └── (contains old sample pages – must **never** be imported or used)


🔹 No JSX, logic, or UI inside app/ files — only imports and exports.
🔹 Only two layouts: panel and ai.
🔹 Each page must have its own sections/[PageName]/ folder.
🔹 The page.tsx and layout.tsx files act as simple routers/importers.
🔹 Objective: Maintain a clean, modular, and scalable structure for large-scale Next.js projects.

🚫 6. Never Do:
❌ Import from .demo_disabled/

7. All UI content generated by the AI — including page titles, buttons, labels, placeholders, error messages, and descriptions — must always be written in English only. 
Do NOT generate any Persian text under any circumstances.
